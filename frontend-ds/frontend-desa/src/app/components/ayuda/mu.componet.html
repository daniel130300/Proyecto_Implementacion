Introducción
El presente documento está dirigido a entregar las pautas de operación del sistema del
local Variedades K&D. Este sistema permite la gestión y manejo de las actividades y
necesidades administrativas, asi como las necesidades para las actividades de los cajeros
del local Variedades K&D. Este sistema requiere del uso de herramientas apropiadas que
nos permitan hacer un seguimiento de los procesos y tareas. Este documento facilitará
la comprensión y uso de las herramientas que ofrece dicho sistema, el usuario podrá ser
capaz de usar el sistema sin ninguna complicación con la ayuda de este manual.
Requerimientos de Hardware y Software
 Espacio disponible de 4GB.
 Tener Windows 7 x64 bits.
 Contar con 4GB de RAM.
 Procesador Intel Core i3-6300u/Ryzen 5 2600g.
 Contar con Postman, MySQL, Sublime text 3, Git Bash y Node.js.
Módulos y Pantallas del Sistema
A continuación, se mostrarán los diferentes módulos y pantallas incluyendo los pasos
para el uso de los controles que posee el sistema del local Variedades K&D, los módulos
y demás serán explicados para que se adquiera el mejor conocimiento de cómo
funcionan los diferentes procesos que el sistema posee.
2
Login
El módulo de Login o Inicio de sesión, es donde el usuario ingresa sus respectivas
credenciales para poder entrar al sistema y así garantizar que el usuario que entra sea
el autorizado para acceder, aumentando así la seguridad e integridad del sistema. Es la
primera pantalla que el sistema mostrará.
<Campo Nombre Usuario>
En este campo de texto se ingresa el nombre de usuario del empleado. Para ello se hace
click en el campo para proceder a escribir dentro de él. Se puede escribir todo tipo de
caracter y número.
<Campo Contraseña Usuario>
En este campo de texto se ingresa la contraseña. Para ello se hace click en el campo para
proceder a escribir dentro de él. Se puede escribir todo tipo de caracter y número.
<Botón Iniciar Sesión>
Botón donde al dar click procederá a verificar las credenciales; si las credenciales fueron
ingresadas correctamente, el usuario será direccionado al menú principal o, si las
credenciales fueron ingresadas de manera incorrecta, aparecerá un mensaje de error
que avisará que ha ingresado mal los datos y no podrá entrar al sistema; todo esto para
aumentar la seguridad y proteger ese acceso.
<Botón ¿Has olvidado tu contraseña?>
Botón que, al hacer click sobre él, el usuario será redirigido al modulo de recuperación
de contraseña, en donde podrá, como su nombre, recuperar su contraseña en caso de
que se le haya olvidado.

Iniciar Sesión el sistema nos mostrará el siguiente mensaje de retroalimentación

Ventas Pendientes de Envío
En esta pantalla el usario podrá visualizar cuáles son las ventas que fueron almacenadas
para un posterior envío, a la vez que podrá actualizar si el envío ya fue efectuado e
imprimir un reporte de las ventas que están pendientes de ser enviadas.
<Barra de Búsqueda>
Barra de búsqueda dinámica que, al posicionarse sobre ella, el usuario podrá realizar
una búsqueda filtrada por los campos que desee.
<Tabla de Ventas Pendientes de Envío>
Tabla en la cual se pueden visualizar el listado de todas las ventas que están
pendientes de envío, también posee un botón de Seleccionar que puede ser
visualizado si utiliza las barras de desplazamiento horizontal.
14
<Botón Seleccionar>
Botón que, al hacer click en él seleccionará la venta deseada para que el usuario pueda
modificar las fechas de envío y de entrega al cliente.
<Botón Reporte>
Botón que, al hacer click en él, generará y descargará automáticamente un reporte en
PDF de todas las ventas pendientes de envío presentes en la tabla.
<Fecha de Envío>
Campo que, al hacer click en él, desplegará un calendario en el cual el usuario podrá
seleccionar la fecha de envío de la venta.
<Fecha de Entrega>
Campo que, al hacer click en él, desplegará un calendario en el cual el usuario podrá
seleccionar la fecha en la que el cliente recibió su pedido, es decir, la fecha de entrega
de la venta.
<Botón Finalizar>
Botón que, al hacer click en él, guardará los cambios realizados en las fechas de envío y
entrega de la venta y que, automáticamente, transferirá la venta al modulo de ventas
pagadas.
Si el usuario no ha seleccionado una venta y procede a presionar el botón Finalizar, el
sistema le mostrará un mensaje de error.
15
En caso de que el usuario escoja una fecha de entrega menor a la fecha en la que fue
enviada la venta, el sistema le mostrará un mensaje de error.
Si las fechas ingresadas son correctas al momento de presionar el botón Finalizar, el
sistema le mostrara un mensaje de que la operación se realizó correctamente.

una venta a un cliente plus, el cual contará con ciertos beneficios a diferencia de un
cliente normal tales como: envio a domicilio, disponibilidad de crédito y derecho a
devoluciones sobre la venta; una vez se haya ingresada la información soliticada en
pantalla el usuario podrá decidir si imprimir o no la factura para el cliente plus,
dependiendo de lo que este desee.

Mantenimiento Empleados
Este módulo se encarga del registro y manejo de los empleados, en este mismo se puede
ingresar un nuevo empleado, modificar uno ya ingresado o removerlo de la lista, el cual
no será eliminado como tal, pero sí será deshabilitado para no aparecer en el listado de
todos los empleados.
84
<Campo Identidad>
En este campo tendremos que ingresar el número de identidad del empleado a registrar,
esta deberá de tener un máximo de total de 13 caracteres, ni más ni menos. Todos
deben de ser números. Este campo es obligatorio.
<Campo Nombre>
En este campo tendremos que ingresar los primeros dos nombres (si se cuenta con
ambos) de la persona que se está registrando. Estos deben de ser ingresados
capitalizados, con su respectiva primera letra en mayúscula y las demás en minúscula.
Este campo es obligatorio.
<Campo Apellido>
En este campo se ingresará los dos apellidos de la persona a registrar, el cual deberán
de ser dos, y ambos con la primera letra en mayúscula. Este campo es obligatorio.
<Campo Teléfono>
En este campo se deberá de ingresar un número telefónico, el cual deberá de ser con el
que se pueda contactar al la persona registrada. Este número deberá de tener ocho
caracteres, todos números.
<Campo Correo Electónico>
Este campo deberá de ser llenado con el correo principal de la persona, pues por medio
de este se le contactará de manera formal. Hay que confirmar que este correo todavía
siga habilitado para contactar al empleado, y de la misma manera hay que percatarse
de que esté ingresado correctamente. Este campo es obligatorio.
85
<Campo Dirección>
Este campo deberá de llenarse con una descripción detallada de la actual ubicación de
la persona que se está registrando, contando también con puntos de referencias los
cuales deberán de ser incluídos para ubicarse de la mejor manera. Este campo es
obligatorio.
<Campo Salario>
En este campo deberá de detallarse el salario por el cual el empleado aceptó el trabajo.
Si el empleado todavía no cuenta con salario determinado, se puede dejar en blanco.
<Campo Nombre de Usuario>
Este campo es donde va un nombre identificativo el cual el empleado usará para iniciar
sesión en la página. Este se deberá de conformar por algún nombre del mismo
empleado, y otras letras para hacerlo único; también se puede ingresar números para
hacer el nombre de usuario mucho más único. Este campo es obligatorio.
<Campo Contraseña>
En este campo se deberá de ingresar una contraseña de mínimo 8 caracteres, la cual
deberá de percatar que el usuario a ingresar puede recordar en todo momento. El
cuadro de texto para ingresar la contraseña se encuentra censurado, pero para poder
ver el contenido de lo que se está escribiendo en la misma, basta con posicionar el
puntero sobre el cuadro y este desvelará la contraseña que se está ingresando. Este
campo es obligatorio.
<Campo Fecha de Nacimiento>
Al dar clic sobre este campo se desplegará un pequeño calendario, en el cual se debe de
buscar la fecha exacta de nacimiento de la persona a registrar. Cabe destacar que este
campo está validado para que no se pueda ingresar ninguna persona que aún sea menor
de edad. Este campo es obligatorio.
<Campo Puesto>
Al dar clic sobre este campo se desplegará una serie de opciones las cuales nos permiten
seleccionar un puesto para el que empleado está aspirando. Este campo es obligatorio.
<Botón Guardar Registro>
Este botón se utiliza para confirmar el ingreso de todos los campos requeridos y creará
un nuevo usuario. Al precionar este botón se sabrá si algún campo está mal y debe de
ser corregido.

Error si el campo "Identidad" se intenta llenar con menos de 13 dígitos: Este error

Error en el campo "Apellido" al tratar de ingresar menos de tres letras: Como el

Error en el campo "Apellido" al tratar de ingresar caracteres numéricos: Este error
